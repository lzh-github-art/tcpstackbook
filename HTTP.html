HTTP：超文本传输协议

HTTP请求报文格式如下。HTTP请求方法常见有GET和POST;URL字段即uri，如https://movie.douban.com/chart 中的/chart;版本号填可写为HTTP/1.1;cr、lf代替回车换行符\r\n。首部行中可填充headers参数，键值间以': '连接，如想表示键为cookie，值为id=1的参数时，可写cookie: id=1。实体主体则是表单数据，对应请求时的payload，键值对之间以'='连接，在需要传输表单数据时，需要设置首部字段Content-Type: application/x-www-form-urlencoded，同时还需设置Content-Length的值为实体长度，如实体为'num=1'时应设置为5。最后，负载数据应以普通字符串的形式组合。

所以，如果想POST请求某个服务器的/index资源时，且表单数据需要传递num时，负载数据可以写为"POST /index HTTP/1.1\r\nContent-Length: 5\r\nContent-Type: application/x-www-form-urlencoded\r\n\r\nnum=1"

现在flaskserver.py文件中会启动一个flask服务器，查看文件，设法请求getrp方法，使返回的消息为'Hello, I get right parameters!'。相关代码编写在httptests中。

static void http(uint16 sport, uint16 dport, int attempts)
{
  int fd;
  // todo：obuf对构造正确的HTTP报文
  char *obuf = "";
  uint32 dst;

  // 10.0.2.2, which qemu remaps to the external host,
  // i.e. the machine you're running qemu on.
  dst = (10 << 24) | (0 << 16) | (2 << 8) | (2 << 0);

  if((fd = connect_tcp(dst, sport, dport)) < 0){
    fprintf(2, "ping: connect() failed\n");
    exit(1);
  }

  for(int i = 0; i < attempts; i++) {
    if(write(fd, obuf, strlen(obuf)) < 0){
      fprintf(2, "ping: send() failed\n");
      exit(1);
    }
  }

  char ibuf[1024];
  int cc = read(fd, ibuf, sizeof(ibuf)-1);
  if(cc < 0){
    fprintf(2, "ping: recv() failed\n");
    exit(1);
  }
  ibuf[cc]='\0';
  printf("rec: %s\n", ibuf);
}
