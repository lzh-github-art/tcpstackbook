TCP：传输控制协议

TCP与UDP都是L4传输层上的协议，UDP无需实现可靠传输，实现相对简单，而TCP需要提供可靠性传输功能，所以协议标准相对繁多，包括三次握手、四次挥手、数据段验证、拥塞控制、计时器器，重传。

在本次实验中，我们将实现一个TCP协议。

校验和

首先，需要先实现校验和计算，见net.c的checksum方法。在UDP中，校验和是可选的，当传入为0时，将不再校验；而在TCP中，校验和是必须的，接收方在检测到校验和不符合时，将抛弃该份数据，这是提供可靠性特点的必要条件。在计算校验和时需要将伪IP头（共12位）拼接起负载数据，然后每16位数据加起来，如果最后剩余8位时，将这8位数据以高8位全为0的方式在合起来。要注意的是，加合过程中会出现数据溢出，所以过程中结果应以32位数据类型进行存储，在最后一步中，将高16位与低16位相加，如果继续存在数据溢出则继续该步骤，直到高16位均为0。编写完成后，运行checksumtests测试是否逻辑正确。

为TCP头填充合适的字段

依据上述TCP头数据格式，在net.c的tcp_output方法中添加tcp头的必要字段：

// sends a TCP packet with necessary parameters
void tcp_output(struct mbuf *m, uint32 dip,
           uint16 sport, uint16 dport, uint32 seqnum, uint32 acknum, uint8 flags, uint16 winsize) 
{
  struct tcp *tcphdr;
  printf("send seqnum= %d, acknum = %d, mbuflen=%d\n", seqnum, acknum, m->len);
  // put the TCP header
  tcphdr = mbufpushhdr(m, *tcphdr);

  // 添加TCP头的必要字段

  tcphdr->checksum = 0; 
  tcphdr->checksum = checksum(htonl(dip), htonl(local_ip), htons(0x0006), htons(m->len), (uint16 *)tcphdr, m->len);

  // now on to the IP layer
  net_tx_ip(m, IPPROTO_TCP, dip);
}

连接建立

TCP端到端之间进行连接建立时需要三次握手, 如下图所示，第一次握手时需要设置SYN标记位，且不能设置ACK位，且需要记录一下序列号x，后续会用到，而确认号无关紧要；第二次握手时，接收方返回的报文必须是确认号为x+1，序列号y接收方自行决定，并且必须要设置标记位ACK和SYN；当客户端收到该报文时，实际上可单方面认为TCP连接已建立，此后都需要设置ACK标记位指示报文有效，此时再通过发送负载数据完成第三次握手。

第二次握手和第三次握手

sysnet.c的sockrecvtcp方法实现接收TCP报文：当发现是第二次握手报文时，应当更新tcp连接状态为TCPS_SYN_RECEIVED，此时客户端可认为连接已建立，可以开始发送数据。第一次发送的这个数据亦为第三次握手的报文，在数据发送后，将状态更新为TCPS_ESTABLISHED。接收到数据时，如果状态为已建立握手，将数据通过mbufq_pushtail放进套接字缓存队列中，应用层会调用read从这个队列中读取数据。
